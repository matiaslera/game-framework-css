El problema de Css

Durante los primeros años de la web, la mayoría de los navegadores no cumplían con las especificaciones estandarizadas de CSS. Fue muy frustrante desarrollar un sitio web que se viera bien en un navegador y un desastre total en otro navegador.
Soportar múltiples navegadores fue una pesadilla. En ese momento, Internet Explorer era el navegador más popular, sin embargo, había implementado mal varias características de CSS, como el modelo de caja . Se crearon varios hacks CSS para solucionar varios problemas con navegadores específicos.
No querrás ver lo malo que fue en versiones anteriores. Todos los principales navegadores se vieron afectados de una forma u otra. Gracias a los esfuerzos del Proyecto de estándares web y las comunidades en línea, la mayoría de los problemas de CSS se han resuelto en los principales navegadores.
Cuando CSS 2.1 se convirtió en una Recomendación propuesta en abril de 2011, tardó unos 13 años en llegar. El nivel 2 de CSS fue la Recomendación del W3C a partir de 1998. La razón por la que tomó tanto tiempo se debe a algunas características de CSS que retrasaron la finalización de la nueva revisión. En cuanto a CSS3, toda la especificación de nivel 2.1 se desglosó en módulos. Cada módulo ahora se desarrolla de forma independiente. El desglose de la especificación ha ayudado enormemente a los proveedores de navegadores a ponerse al día más rápido con las últimas recomendaciones a nivel modular.
Hoy, tenemos algunos módulos en borrador en el nivel 4. Otros están en el nivel 3. Si se agrega una nueva característica CSS, se convierte en el nivel 1. Puede visitar la página de trabajo CSS del W3C para ver el estado actual de cada módulo. También puede visitar css3test.com para verificar qué tan bien su navegador admite diferentes módulos CSS.

Historia de framewoks

Alrededor de mediados de la década de 2000, las bibliotecas y marcos CSS comenzaron a surgir. Estos marcos introdujeron un sistema de cuadrícula para ayudar a los diseñadores web a diseñar su contenido. Los marcos soportaban la mayoría de los navegadores (un dolor de cabeza menos para los diseñadores). Algunos de los primeros marcos CSS incluyeron Blueprint , 960 , YUI Grids y YAML .
Cuando los fabricantes de teléfonos móviles comenzaron a enviar dispositivos con navegadores web, no hubo mucho esfuerzo para rediseñar los sitios web para dispositivos móviles. Los navegadores móviles eran capaces de reformatear las páginas web para que se ajustaran a las pantallas pequeñas.
Con el paso del tiempo, el hardware colocado en los teléfonos mejoró enormemente. En 2006, Opera Mobile se convirtió en el primer navegador móvil en pasar la prueba ACID2 . Safari fue el primer navegador de escritorio en pasar la prueba.
En 2007, Steve Jobs anunció el primer iPhone. Tenía la pantalla más grande en ese momento. Pronto se lanzaron dispositivos Android con pantallas similares o de mayor tamaño. La popularidad de los teléfonos inteligentes y la navegación móvil condujo a una gran aceptación del uso de Internet. Pronto, las empresas comenzaron a darse cuenta de la importancia de tener que diseñar sus sitios web para pantallas móviles.
En 2011, Twitter lanzó el marco CSS Bootstrap como un proyecto de código abierto. En 2012, Bootstrap 2 se lanzó con un sistema de diseño de cuadrícula sensible de doce columnas junto con muchas otras características nuevas. En 2013, se lanzó Bootstrap 3 que había rediseñado componentes y tenía una filosofía de diseño móvil primero.
Bootstrap versión 4, lanzada en agosto de 2017, agregó soporte para Sass y Flexbox . Bootstrap es actualmente el marco CSS más popular. Es más popular entre los desarrolladores que desean crear prototipos rápidamente de una interfaz web de gran apariencia.
En octubre de 2011, ZURB abrió su framework CSS y lo lanzó como Foundation 2.0 . Presentaba una grilla receptiva y soportaba Sass. También tenía una extensa lista de elementos de interfaz de usuario prediseñados. Foundation es más popular entre los diseñadores que desean un control completo sobre la personalización de su sitio. Foundation también viene con plantillas para correos electrónicos.
